📖 项目简介

UtaSync 致力于解决传统 AI 字幕工具在处理“复杂音乐与现场场景”时面临的三大史诗级痛点：

乐器轰炸致盲：架子鼓和失真吉他盖过人声，导致 AI 听写彻底变成乱码或静音。

长间奏字幕霸屏：长达数十秒的纯音乐间奏，导致上一句字幕停留在屏幕上无法消散。

机翻空耳与格式崩坏：传统机翻不结合上下文与官方歌词，频发“空耳”错别字，且长句排版拥挤不堪。

本项目将 Demucs (人声分离)、Faster-Whisper (极速打轴) 与 LLM 大模型 (智能同传与排版) 深度融合。辅以极度强壮的防崩溃工程底座，提供从音视频导入到精美双语字幕产出的“一键式”体验。

✨ 核心技术特性 (Core Features)

🎸 1. 手术刀级提音与无缝切片 (Audio Processing)

极限剥离：无惧千人合唱与重金属斗琴，通过 Demucs 完美剥离纯净人声，根除 Whisper 纯乐器幻觉。

动态防爆内存：首创针对 2 小时+ 蓝光 Live 的**「无缝切片处理机制」**。音频自动切割分段进入显存，分离后再无痕拼接，彻底告别超长视频导致的 OOM (内存溢出) 崩溃。

⏱️ 2. 智能打轴与防霸屏 (Whisper VAD Tuning)

双模式声学模型：内置【Live 现场】（高容忍，防换气切碎）与【电台访谈】（低容忍，高频切分）双重参数集。

8秒物理熔断：强制拦截长间奏幻觉，单句字幕停留最多不超过 8 秒，完美解决超长吉他 Solo 导致的字幕霸屏错位。

🧠 3. 大模型“开卷”翻译与同传 (LLM Translator)

官方歌词强制注入：支持直接加载网易云/QQ音乐官方歌词。大模型手握标准答案进行校对，误听率降至 0%。

无缝 MC 切换：遇到无歌词的互动环节（Talking/MC），自动无感知切换至同声传译盲翻模式。

强迫症级排版：代码层物理锁死时间轴。强制大模型产出严格的【1行日文 + 1行中文】，严禁违规合并或换行。

🛡️ 4. 工业级容错与安全防线 (Fault Tolerance)

断网/限流续传：内置 JSON 进度缓存。遭遇 API 429 限流会自动阶梯休眠；断电重启也能从中断的批次满血复活。

妥协放行机制 (Graceful Degradation)：面对大模型连续报错的“毒药乱码数据”，系统会在第 5 次重试时强行通关并隔离错误，誓死保卫主程序不崩溃。

⚙️ 快速上手 (Quick Start)

1. 环境准备

请确保您的电脑已安装 Python 3.10+，并且必须在系统环境变量中配置好 FFmpeg。

# 1. 克隆本仓库
git clone [https://github.com/你的用户名/UtaSync.git](https://github.com/你的用户名/UtaSync.git)
cd UtaSync

# 2. 安装 Python 核心依赖库
pip install -r requirements.txt
pip install customtkinter


(💡 性能提示：为获得极其百倍的打轴速度，请确保您的电脑配备 NVIDIA 显卡并已正确安装 CUDA 与 cuDNN。)

2. API Key 配置与安全提示

UtaSync 支持配置多家大模型（如 SiliconFlow 硅基流动、Google Gemini 等）。
⚠️ 安全警告：请勿在代码中硬编码您的 API Key 并上传至公开仓库！建议在 V2.0 桌面端的 UI 界面中手动填入，或使用环境变量。

3. 一键起飞 🚀

👉 启动 V2.0 桌面端图形界面（强烈推荐）：

python app.py


特性：支持拖拽导入、免文件歌词粘贴、多线程防假死、实时进度控制台。

👉 启动 V1.0 极客命令行版：

python main.py

UtaSync/
