# UtaSync 项目架构分析

## 1. 系统整体架构

UtaSync 是一个专为日语 Live 视频设计的自动字幕生成与翻译系统，采用模块化设计，主要分为两个核心处理阶段：打轴引擎和翻译引擎。

**系统架构图**：
```
┌─────────────────┐     ┌──────────────────────┐     ┌──────────────────────┐
│  视频输入文件   │────>│  LiveSubtitleGenerator  │────>│  SubtitleTranslator   │
└─────────────────┘     └──────────────────────┘     └──────────────────────┘
        │                       │                               │
        │                       ▼                               ▼
        │               1. 提取音频                           1. 读取参考歌词
        │               2. 分离人声                           2. 解析SRT文件
        │               3. 生成时间轴字幕                      3. 批量翻译
        │               4. 整理输出文件                       4. 输出最终字幕
        │                       │                               │
        ▼                       ▼                               ▼
┌─────────────────┐     ┌──────────────────────┐     ┌──────────────────────┐
│   原始视频      │     │  纯净人声 + 原文字幕   │     │  双语/纯中文字幕      │
└─────────────────┘     └──────────────────────┘     └──────────────────────┘
```

## 2. 主要技术栈

| 类别 | 技术/库 | 用途 | 来源 |
|------|---------|------|------|
| 后端语言 | Python | 核心开发语言 | main.py, core/ |
| 语音识别 | openai-whisper | 语音转文字基础库 | requirements.txt |

## 3. 核心模块划分

### 3.1 LiveSubtitleGenerator (打轴引擎)
- **职责**：从视频中提取音频，分离人声，生成带时间轴的原文字幕
- **核心功能**：
  - 音频提取：使用 FFmpeg 从视频中提取音频
  - 人声分离：使用 Demucs 高精度模型分离人声
  - 语音识别：使用 Faster-Whisper 模型生成时间轴字幕
  - 文件管理：整理输出文件，优化目录结构

### 3.2 SubtitleTranslator (翻译引擎)
- **职责**：将生成的原文字幕翻译为中文（双语或纯中文）
- **核心功能**：
  - 参考歌词加载：智能寻找并加载参考歌词
  - SRT 解析：解析原始字幕文件
  - AI 翻译：调用大模型 API 进行翻译和纠错
  - 格式处理：处理不同输出模式的格式要求

### 3.3 主程序 (main.py)
- **职责**：提供用户交互界面，协调各模块工作
- **核心功能**：
  - 用户输入处理：获取视频路径和用户选择
  - 配置管理：管理 API 配置和模型选择
  - 流程控制：协调打轴和翻译流程
  - 错误处理：处理异常情况

## 4. 模块间通信方式

- **同步调用**：主程序直接实例化并调用核心模块的方法
- **返回值传递**：模块通过返回值传递处理结果（如 srt_path）
- **文件系统**：模块间通过文件系统传递数据（如生成的 SRT 文件）

## 5. 数据流向

```
视频文件 → LiveSubtitleGenerator → 提取音频 → 分离人声 → 生成原文字幕 (SRT) → SubtitleTranslator → 读取参考歌词 → 翻译排版 → 输出最终字幕
```

**详细流向**：
1. **输入阶段**：用户提供视频文件路径
2. **打轴阶段**：
   - 提取音频：从视频中提取 WAV 音频
   - 分离人声：使用 Demucs 分离纯净人声
   - 语音识别：使用 Faster-Whisper 生成带时间轴的字幕
   - 输出原文字幕：保存为 SRT 文件
3. **翻译阶段**（可选）：
   - 加载参考歌词：智能寻找并加载参考歌词
   - 解析 SRT：读取原始字幕文件
   - AI 翻译：调用大模型 API 进行翻译和纠错
   - 输出最终字幕：保存为双语或纯中文字幕

## 6. 部署架构

### 6.1 开发/测试环境
- **硬件要求**：
  - 推荐配备 NVIDIA GPU（支持 CUDA）以获得最佳性能
  - 至少 8GB 内存
  - 足够的存储空间用于模型和输出文件
- **软件要求**：
  - Python 3.10+
  - FFmpeg
  - 依赖库：openai-whisper, demucs, ffmpeg-python
- **环境配置**：
  - 国内用户建议设置 HF_ENDPOINT 为 https://hf-mirror.com 以加速模型下载
  - API Key 配置：在 main.py 中配置大模型 API Key

### 6.2 生产环境
- 本项目为本地工具，无需服务器部署
- 可通过打包工具（如 PyInstaller）生成可执行文件，方便用户使用

## 8. 架构设计中的关键决策和考虑因素

### 8.1 性能优化
- **GPU 加速**：优先使用 GPU 进行语音识别，提升处理速度
- **批处理**：翻译阶段采用批量处理，减少 API 请求次数
- **缓存机制**：对已处理的纯净人声进行缓存，避免重复处理

### 8.2 用户体验
- **清晰的目录结构**：按视频名称组织输出文件，音频和字幕分离存放
- **友好的交互界面**：提供详细的选项说明和进度反馈
- **智能默认值**：为常用选项提供合理的默认值，减少用户决策

### 8.3 容错处理
- **API 重试机制**：处理 API 频率限制，自动重试
- **错误捕获**：捕获并处理各种异常情况，提供清晰的错误信息
- **降级策略**：当 GPU 不可用时，自动降级使用 CPU

### 8.4 可扩展性
- **模块化设计**：核心功能模块化，便于扩展和维护
- **多模型支持**：支持多种 Faster-Whisper 模型和大模型 API
- **配置灵活性**：通过配置文件管理 API 和模型参数

### 8.5 质量保证
- **参考歌词**：使用参考歌词提高字幕准确性
- **智能排版**：针对不同输出模式优化字幕排版
- **时长熔断**：防止字幕过长影响观看体验

## 9. 总结

UtaSync 是一个设计精巧的视频字幕生成与翻译工具，特别针对日语 Live 视频优化。它通过模块化设计、高性能处理和智能排版，为用户提供了一套完整的字幕解决方案。系统充分利用了现代 AI 技术，结合语音识别和大语言模型，实现了从视频到高质量字幕的自动化流程。

该架构设计考虑了性能、用户体验和可扩展性等多个方面，为用户提供了一个高效、易用的字幕工具。同时，系统的模块化设计也为未来的功能扩展和性能优化预留了空间。
